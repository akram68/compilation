   - Compile all C files together

3. If you get path errors, update the Makefile with the correct paths to win_flex.exe and win_bison.exe

HOW IT WORKS NOW:
==================

Your compiler now has full code generation! When you run:

A = [1, 2; 3, 4];
B = [5, 6; 7, 8];
C = A * B + A';
print(C);

The compiler will:
1. Parse and validate the syntax (lexical & syntax analysis)
2. Check variable declarations (semantic analysis)
3. CREATE actual matrix structures in memory
4. PERFORM the matrix multiplication: A * B
5. PERFORM the matrix transpose: A'
6. PERFORM the matrix addition: (A*B) + A'
7. STORE the result in variable C
8. PRINT the actual computed result!

WHAT'S NEW:
============

1. Matrix Structure: Each matrix stores its dimensions and data
2. Matrix Operations:
   - Addition: element-wise addition with dimension checking
   - Subtraction: element-wise subtraction
   - Multiplication: proper matrix multiplication (AxB requires A.cols = B.rows)
   - Transpose: flips rows and columns

3. Code Generation: The compiler now:
   - Creates actual matrix objects during parsing
   - Performs computations as expressions are evaluated
   - Stores results in a symbol table
   - Can retrieve and display computed values

4. Error Checking:
   - Dimension compatibility for operations
   - Undefined variable detection
   - Matrix row size validation
